Product Diagnosis · web.token.im

Home of Tokens
诊断 · 市场反馈 · 重设计策略

imToken 推出的 Web 版钱包 web.token.im 定位为「Home of Tokens」，以 Passkey + AA 账户 + Send-by-Link 为核心差异化。上线后用户量有限、首页对未注册用户几乎没有价值传递与引导。本报告用 PM / Designer / CEO 三个视角切入，给出一套可执行的 onboarding 内容与设计策略。

评估日期 2026-04-24 评估对象 web.token.im（Desktop 1512×949） Beta 状态 仍在持续迭代 支持网络 Arbitrum (主)

一、首屏体验快览

Desktop 首次访问时的实际观察：页面加载有约 3 秒白屏，随后进入一个「漂浮气泡」可视化主视觉，没有滚动内容，没有对未注册用户的 CTA。

正文与背景对比度（WCAG 风险）

页面结构（实际观察）

  • Header：Logo / Search / My Holdings / My Collections / Account
  • Hero 文案：「Home of Tokens — Digital assets under control」
  • 主视觉：一组漂浮气泡（Bluechip Stables、Yield Stables、Stock Tokens、Arbitrum Ecosystem、DeFi Stables、Regulated Stables…），中央是「My Holdings」气泡
  • 初次访问弹出 Welcome modal（Passkey 介绍 + Send by Link），可跳过
  • 无 Footer、无产品介绍、无信任背书、无 FAQ

产品官方定位（来自官网/帮助中心）

  • 非托管、浏览器原生，无需下载、无需助记词
  • Passkey 生物识别登录，抗钓鱼
  • AA 账户 + Paymaster：可用 ERC-20 付 gas
  • Send by Link：红包式分享代币，接收方无需先有钱包
  • Token 分类浏览：稳定币、收益型、股票 Token、DeFi、RWA
定位 vs 表达的断裂：官方叙事清晰雄心勃勃，首页却几乎没有把这五件事讲给访客。

二、市场与社媒反馈

综合 Trustpilot、G2、SourceForge、App Store / Play Store、Medium、CryptoCompare、TradersUnion 及 Arbitrum 官方博客等公开信息汇总。

🟥 负面声音（集中点）

  • Trustpilot 对 token.im 的整体评分约 1.7 / 5，负面占比显著
  • 英文文案存在拼写/语法/口径问题，"made in China" 的感知被放大
  • 历史上被第三方诈骗站点冒用与 WalletConnect 连通性问题
  • 部分交易/Swap 场景出现空白页、无法提币的反馈

🟩 正面声音

  • 多链 & 入门友好，50+ 网络、预置主流 RPC
  • Passkey + AA 被技术圈认可为「下一代 UX 方向」
  • Devcon 7 Bangkok 上作为赞助商 + 咖啡活动首次发布，技术媒体给予正面报道
  • Help Center 文章帮助度：Quickstart 18/20，公告 8/9，说明用户愿意读

🟨 结构性挑战

  • Web3 钱包品类高度拥挤：MetaMask、Phantom、Rainbow、Rabby、Zerion、OKX Wallet…
  • Passkey 叙事已被 Coinbase Smart Wallet、Argent、Safe{Core} 等占用
  • imToken 在西方市场的品牌认知偏低，中国/亚洲认知强但受监管影响
  • Beta 标签 + 仅 Arbitrum，体感「玩具感」重、可信度不足
imToken 在 Medium 上自述：「Tokens 将像电力和互联网一样，渗透到日常生活里。」 — imToken Medium, "Exploration of the Next-Gen Web3 Wallet UX"

三、PM / Designer / CEO 三视角评估

同一个首页，不同角色会看到不同问题。下面是三个视角下最扎心的判断。

📊
Product Manager
关心转化与漏斗
P0 严重

漏斗断点：访客 → 注册 之间缺少驱动力

首屏既无价值主张、也无 CTA，对未注册用户等于「空货架」。passkey / AA / send-by-link 这些差异化被藏在 modal 与帮助中心里。

高流失
P0 严重

信息架构错位：「My Holdings」对新用户毫无意义

把「我的持仓」放在全站最显眼位置，对未登录用户是 0 数据；Stock Tokens / Yield Stables 等标签没有教学，纯靠猜。

高认知成本
P1 重要

Beta 标签 + 仅 Arbitrum 没有 framing

技术限制没有变成产品叙事（如 "Start on Arbitrum, more chains in 2026"），用户感知是「半成品」。

信任风险
P1 重要

缺失激活指标与内嵌遥测

看不出有可度量的 activation 事件（create passkey、first receive、first send-by-link）被用来驱动新手引导分支。

留存风险
🎨
Designer
关心层级与可读性
P0 严重

视觉层级缺失，气泡把注意力吃光

标题、气泡、入口按钮争夺同一个视觉中心；大量低饱和浅灰文字 < 4.5:1 对比度，标签几乎不可读。

可访问性
P0 严重

没有一个主 CTA 按钮

首屏没有任何 Filled Button。Create Wallet / Get Started 这类关键入口完全缺席，靠 Header 里的 Account 图标承担注册动作，极易错过。

转化设计
P1 重要

主视觉动效「装饰大于叙事」

漂浮气泡漂亮但不传达产品能力，既不是真实数据、也没有引导用户点哪一个；对新用户是 "看不懂"。

叙事弱
P2 优化

首屏白屏时间长 + 无骨架屏

~3 秒白屏期间无任何 skeleton / 静态文案兜底，品牌印象从「慢」开始。

性能感知
👑
CEO
关心定位与增长
P0 严重

「Home of Tokens」没有落成一个可感知的品牌叙事

口号停留在 Slogan 层面，没有被翻译成 3 句让西方记者也能引用的价值主张。对比 Coinbase Smart Wallet、Rainbow、Phantom 都有清晰一句话定位。

品牌
P0 严重

GTM 漏斗没闭环：Devcon 曝光没接住

Bangkok Devcon 7 发布，但首页没有「Claim free token」「From Devcon」专属入口接住那波流量；活动结束，流量回落。

增长
P1 重要

产品定位与商业模型暧昧

到底是「入门级钱包」「稳定币/RWA 消费入口」还是「AA 账户基础设施」？首页同时塞了三个叙事，结果谁都讲不透。

定位
P1 重要

区域品牌感拖累国际化

中文风格 + 英文翻译瑕疵叠加过往负面评价，让海外用户不容易给信任。首页需要主动承担「国际化去标签化」的工作。

信任

四、策略重设计：从 Visualization 页 → Onboarding 页

核心原则：首页对未注册用户是「说服 + 转化」，对已登录用户才是现在的「资产/分类浏览」。不要再用一个页面同时服务两类人。

定位一句话（建议）

「在浏览器里，用指纹就能拥有一个钱包；用链接就能把代币发给任何人。」

相比现在的 "Digital assets under control"，更具体、更动词、更可验证。

访客状态分层

  • Anonymous 未登录：主打 Hero + 3 大能力 + 30 秒 Demo + CTA
  • Just Created 刚注册：引导完成首次接收/Send-by-link/Explore
  • Returning 老用户：才进入当前的 Tokens 浏览器视图

4.1 Content 蓝图（Above-the-fold → 滚动）

区块内容目的
Hero 标题："Your wallet, without the wallet part."
副标题：Passkey 登录，无需助记词；发送代币只需一条链接。		 30 秒内让访客看懂产品 & 差异化
主 CTA 「Create wallet in 10 seconds」+ 「Send yourself $1 to try」 低阈值首次激活
3 大能力 ① Passkey security (no seed phrase) ② Gas in USDT (Paymaster) ③ Send-by-Link (red packet for tokens) 把埋在帮助中心的差异化显性化
30s Demo 一个循环动效 / 短视频：指纹 → 钱包开通 → 贴一个链接 → 朋友领到 USDT 让访客"相信"而不是"理解"
Token 探索预览 保留现在的气泡视觉，但作为 "Explore the Token Universe" 的 preview，附带真实价格 & 点击进详情 复用资产、且让视觉有功能性
信任带 Arbitrum / Ethereum / Tokenlon 等生态 logo，审计报告、passkey WebAuthn 说明 对冲 "Made in China" 偏见
Beta 叙事 清晰 roadmap：Q2 加入 Optimism / Base，Q3 加入 Linea / zkSync 等 把 Beta 限制变成进度感
FAQ + Footer Passkey 是什么？丢了手机怎么办？为什么只支持 Arbitrum？ 扫除首次尝试前的阻力问题

4.2 Onboarding 流程（5 步最小可行路径）

Step 01

Land

Hero 内嵌 "Create in 10s" 按钮，无需跳转

Step 02

Passkey

起名 + 指纹确认，强调「没有助记词」安心感

Step 03

First Taste

赠送 0.5 USDT gas 额度 / 触发一次真实 on-chain 体验

Step 04

Send-by-Link

引导生成一条给朋友的 red-packet 链接（病毒增长循环）

Step 05

Habit Hook

完成后展示个性化的 Token Collections，绑定 push/邮件

这套五步是一个同时承担激活与增长的 loop：Step 4 的 Send-by-Link 让每一个新用户都是潜在的拉新节点。

4.3 Design 系统调整要点

Typography

正文对比度 ≥ 7:1；主标题 56–72px；小标签最低 14px；全站字号阶梯化。

Color

保留 Token 渐变蓝紫色作为品牌色，但加入明确的 primary button fill、error、success 语义色。

Motion

气泡动效移到 "Explore" 区块，做成「围绕你的资产」的呼吸感，而不是占住首屏。

Components

新增：Empty Hero、Feature Card、Demo Player、Trust Bar、Roadmap、FAQ Accordion。

States

首页必须有三态：Anonymous / Onboarding / Returning，分别走不同布局。

Performance

静态 Hero 首屏 < 1s；气泡动画懒加载；加 skeleton 过渡。

六、一句话总结

现在的首页在对老用户讲话，却把 99% 的新访客当成已经相信你的人。把首页重新当成「产品白皮书的 10 秒版」，让 Passkey、Gas 抽象化、Send-by-Link 这三件事显性化，Home of Tokens 才真正成立。
