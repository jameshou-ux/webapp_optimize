02 背景

首页具备很强的产品潜力，但价值表达、活动运营与发现流程仍未对齐

Spatial View 有差异化，但信息支撑不足。List View 更适合运营，但辨识度较弱。首页已有注册入口，但 AA 的轻量化优势仍未被清楚表达。

03 问题

当前门户问题

在审计、现网检查与原型探索中，反复出现的核心问题集中在这四点。

01

价值表达不够快

用户先看到 token 视觉，再去猜 imToken Web 到底解决什么问题。

02

活动承载能力弱

List View 有海报和精选位，但 CTA 与活动逻辑都偏弱；Spatial View 也缺少对应的活动层。

03

Spatial 与 List 能力不均衡

一个更适合品牌表达，一个更适合运营承载，但两者还没有形成一个统一的首页系统。

04

信息与 onboarding 都偏薄

Collection 缺少足够的点击动机，轻量 passkey onboarding 也没有被塑造成明确优势。

04 框架

三个潜在改进方向

聚集低成本改造，不引入新功能和依赖。

方向 1

发现优先

最直接回应 collection 缺乏动机、campaign 理解不足的问题。

方向 2

融合 Spatial/List

最直接回应 Spatial 与 List 割裂的问题，用统一外壳和 collection 侧栏建立连续性。

方向 3

onboarding 优先

最直接回应 onboarding 表达不清和转化信心不足的问题。

05 方向 1

发现优先更便于探索

让首页首先成为更强的 token 发现与 campaign 承载界面。

View
P

对应问题：用户没有足够理由点进 collection，campaign 也缺少明确理解。

R

设计回应：更丰富的 collection 卡片、更清晰的 campaign 模块，以及进入深层页面前更强的决策层。

W

价值：提升扫读层的理解效率，让发现行为更容易转化为下一步动作。

06 方向 2

融合 Spatial/List 与持续 collection 侧栏

当前 V2 方向：让 Spatial 和 List 成为同一个首页系统的两种视角，以 collection 侧栏作为连续性的核心。

View
  • 对应问题：Spatial 与 List 当前既不均衡，也不连续。
  • 设计回应：统一外壳、统一选中状态、统一侧栏，同时覆盖两种模式。
  • 模式切换保留上下文，而不是把用户送去另一个割裂的体验。
  • 进入页面时可默认选中活动 hub 或精选 collection。
  • 这是当前 V2 原型中最有价值的洞察。
07 方向 3

onboarding 优先

让首页首先成为轻量激活与 onboarding 入口。

View
P

对应问题：首页没有把轻量 AA onboarding 讲清楚。

R

设计回应：更强的一屏 CTA、passkey 信心提示、能力提示卡，以及 15 秒 onboarding demo。

W

价值：提升转化信心，并把 onboarding 变成可见的产品优势。
