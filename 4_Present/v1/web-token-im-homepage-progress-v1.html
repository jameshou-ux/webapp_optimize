02 Context

The homepage has strong product potential, but value, campaign operation, and discovery flow are not aligned yet

Spatial View is differentiated but under-informative. List View is more operable but less distinctive. Onboarding entry exists, but the AA advantage is still under-explained.

03 Issues

Current portal gaps

Four core problems consistently showed up across audit, live review, and prototype framing.

01

Value is not clear fast enough

Users see token visuals before they understand why imToken Web matters.

02

Campaign surface is weak

List View has poster-like modules, but weak CTA and weak campaign logic. Spatial View lacks an equivalent layer.

03

Spatial and List are uneven

One is stronger for brand, the other is stronger for operation. They do not yet work as one homepage system.

04

Information and onboarding are too thin

Collections lack enough reason to click, and lightweight passkey onboarding is not framed as a clear advantage.

04 Framework

Three parallel directions, not three visual variations

Each direction is a different strategic response to the homepage problem, not one shared wireframe with different emphasis.

Direction 1

Discovery-first

Best response to weak collection motivation and weak campaign understanding.

Direction 2

Merged Spatial/List

Best response to the split between Spatial and List, using one shared shell and collection sidebar.

Direction 3

Activation-first

Best response to weak onboarding clarity and weak conversion confidence.

05 Direction 1

Discovery-first

Make the homepage primarily a stronger token discovery and campaign surface.

R

W

06 Direction 2

Merged Spatial/List with persistent collection sidebar

Current V2 direction: Spatial and List become two lenses of one homepage, with the collection sidebar as the continuity layer.

  • Problem addressed: Spatial and List currently feel uneven and disconnected.
  • Response: one shared shell, one selected collection state, one sidebar across both modes.
  • Mode switch preserves context instead of sending users to a different experience.
  • Campaign hub or featured collection can be auto-selected on open.
  • This is the strongest current V2 prototype insight.

07 Direction 3

Activation-first

Make the homepage primarily a lightweight activation and onboarding entry point.

P

Problem addressed: the homepage does not clearly communicate lightweight AA onboarding.

R

Response: stronger first-viewport CTA, passkey reassurance, capability cues, and 15s onboarding demo.

W

Why it matters: improves conversion confidence and turns onboarding into a visible product advantage.

08 Readout

Early readout from the prototype work

The current exploration already points to where each direction is strongest.

What looks strongest so far

  • Richer information clearly improves collection motivation.
  • The merged shell is promising for resolving the current mode split.
  • Activation-first framing is strongest when the goal is to explain the product advantage quickly.

What still needs review

  • Which problem should the homepage solve first: discovery, continuity, or activation?
  • How much homepage real estate should go to onboarding versus exploration?
  • How strongly should Spatial differentiation be protected in the final direction?
