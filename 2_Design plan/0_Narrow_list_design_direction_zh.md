# 方向收窄设计指导文档

日期：2026-04-28
更新：2026-04-29
来源：`web-token-im-homepage-design-optimization-directions-2026-04-28.md`、`0_Project Context/imtoken-web-evaluation.html`
目的：定义三条并行的首页方向，用于原型生成阶段；已更新以反映当前 V2 合并版首页方向

---

## 如何使用本文档

以下三个方向是**并行**的概念路径。

它们不是递进关系，不应叠加使用。

在原型生成阶段，输出应为：

- 3 个独立的原型；
- 每个原型有各自的首页定位；
- 每个原型做出不同的布局和信息架构选择；
- 除非该方向明确要求，否则各原型不应共用同一套全局导航加子导航结构。

重要规则：

- 共同的业务约束可以在三个方向中保持一致；
- 共同的页面布局结构不应被强制套用于所有方向；
- V2 合并版 Spatial/List 首页（含持久化集合侧栏）仅属于方向二，在本阶段不适用于其他方向。

---

## 三个方向共同遵守的约束

所有方向都应遵守以下约束：

- 不使用助记词相关的表述；账户抽象和 Passkey 注册是核心产品假设；
- 保持注册流程轻量化；
- 不依赖 AI 洞察功能；
- 不依赖新的个性化推荐引擎；
- 优先低成本地使用现有 API 和已有内容字段；
- 活动（Campaign）内容必须可由市场团队独立运营；
- 集合卡片可展示已有的丰富信息，如 Token 数量、所在链、代表性代币、价格或 24 小时涨跌幅；
- 当前首页信息密度过低，应予以改善；
- 首页应激励用户探索并深入导航。

---

## 方向定位概述

以下三个方向应被视为对同一首页问题的三种不同解答：

1. **方向一**：将首页作为更强的代币发现和活动展示入口。
2. **方向二**：将首页作为 Spatial/List 合并式发现框架，配合集合侧栏。
3. **方向三**：将首页作为轻量化激活与引导注册的入口。

三者在业务层面相关，但在原型输出中应保持视觉和结构上的独立性。

---

## 方向一：发现优先型首页

### 核心意图

将首页的核心功能定位为代币发现、活动理解和集合点击动机。

本方向应能回答：

- 为什么我应该打开这个集合？
- 当前有哪些重点内容？
- 在注册账户之前，我能了解到哪些有用的代币信息？

### 核心设计动作

- 利用现有元数据和代币信号，呈现更丰富的集合卡片；
- 活动模块包含标题、描述、关联集合/代币、CTA 及状态（如有）；
- 强化集合详情页作为决策层的作用；
- 在浏览层面提供更清晰的集合价值说明。

### 本方向强调什么

- 集合的可理解性；
- 活动信息的清晰度；
- 浏览层面的代币动机；
- 给用户更充分的点击和探索理由。

### 本方向不需要突出什么

- 以大型注册引导 Hero 区作为首要内容；
- 以"市场 / 我的持仓"Tab 拆分作为首页框架；
- 顶层 Tab 加模式子导航的叠加导航模型；
- 以投资组合所有权作为首页主要信息。

### 原型指导

方向一可以使用：

- 一个首页发现展示面；
- 一种主导浏览模式；
- 活动驱动的模块；
- 更丰富的编辑/市场卡片。

方向一不应看起来像是在方向二的合并框架上套了一层不同的样式。

---

## 方向二：Spatial/List 合并型首页

### 核心意图

将首页定位为一个统一的发现框架，Spatial 视图和 List 视图是对同一集合体系的两种浏览方式。

本方向应能回答：

- 如何让 Spatial 和 List 感觉是同一个首页，而不是两个独立体验？
- 用户切换模式时，如何保留上下文？
- 在浏览过程中，如何保持集合信息的可见性？

### 核心设计动作

- 一个 Spatial 和 List 共用的首页框架；
- 紧凑的 Spatial/List 模式切换控件；
- 持久化的集合侧栏或选中面板，作为核心决策层；
- 页面加载时自动选中活动 Hub 或精选集合；
- 同一集合对象在两种模式下均有渲染；
- List 模式使用与 Spatial 相同的侧栏逻辑，而非独立的弹层/详情页模式；
- "我的持仓"可以作为次级 Tab 存在，但合并的市场框架才是核心。

### 本方向强调什么

- 模式切换之间的连续性；
- 不离开首页即可理解集合内容；
- Spatial 作为主探索视图；
- List 作为相同内容的紧凑浏览模式；
- 侧栏作为浏览与行动之间的桥梁。

### 本方向必须包含的内容

- 共用一个框架的 Spatial 和 List 视图；
- 跨模式持久存在的集合侧栏或选中面板；
- 将侧栏作为一级元素而非临时详情弹层的布局；
- 侧栏内有足够空间容纳活动背景、代币行、CTA 和账户引导信息。

### 原型指导

方向二应遵循现有素材中 V2 合并首页的模式：

- 主浏览区与集合侧栏约为 65/35 的比例分割；
- 一种模式下为 Spatial 画布；
- 另一种模式下为 List 网格或行；
- 两种模式下侧栏均显示同一选中集合的面板；
- 页面打开时自动选中活动 Hub；
- 切换模式时保留已选中的集合上下文。

只有方向二可以使用合并框架加集合侧栏的模式。

---

## 方向三：激活优先型首页

### 核心意图

将首页定位为帮助用户理解账户注册有多轻量的入口，并引导他们完成第一个操作。

本方向应能回答：

- 这是什么产品？
- 注册有多简单？
- 我现在为什么应该创建账户？
- 注册完成后接下来会发生什么？

### 核心设计动作

- 清晰的账户创建主 CTA；
- 简洁的账户抽象/Passkey 安全感说明；
- 3 个能力点展示：Passkey 账户、Gas 抽象、链接发送；
- 约 15 秒的引导注册演示；
- 注册后引导进入有意义操作的流程；
- 一至两个精选活动或集合，作为行动场景的依托。

### 本方向强调什么

- 激活信心；
- 降低设置门槛带来的焦虑；
- 首次操作的清晰路径；
- 将引导注册作为一个流程而非教育页面。

### 本方向不需要突出什么

- 密集的集合分类体系；
- 宽泛的市场浏览框架；
- 以"市场 / 我的持仓"作为首页主结构；
- 将 Spatial 和 List 作为首页的两个并列模式。

### 原型指导

方向三可以使用：

- 更强的首屏 CTA 区块；
- 可见的演示/信任证明时刻；
- 激活内容之下或旁侧的辅助浏览模块；
- 从首页到账户创建的更短路径。

方向三不应演变为带有持久侧栏的合并发现框架。其核心任务是激活用户。

---

## 方向隔离规则

为避免原型重叠，请遵守以下明确规则：

- 方向一不应继承合并式 Spatial/List 框架及持久集合侧栏。
- 方向三不应继承合并式 Spatial/List 框架及持久集合侧栏。
- 方向二可按需使用"市场 / 我的持仓"，但其真正的差异化特征在于合并的 Spatial/List 框架和侧栏的连续性。
- 方向一应将丰富的活动和集合理解放在首位，而非注册引导结构。
- 方向二应将模式连续性和选中状态的可理解性放在首位，而非以 Hero 区驱动的激活流程。
- 方向三应将引导注册的清晰度放在首位，而非宽泛的发现密度。

---

## 共用的低成本内容机会

任何方向均可使用以下内容（如已有数据支持）：

- 价格及 24 小时涨跌幅；
- 链/网络标签；
- 代币数量；
- 代表性代币名称；
- 活动标签和 CTA；
- 来源/类型标签；
- Passkey/无助记词说明文案；
- Gas 抽象说明；
- 链接发送说明；
- 轻量化的信任/状态提示。

这些是共用的内容机会，不能以此为据要求三个方向共用同一布局。

---

## 工作总结

下一个原型阶段应生成 3 条并行的首页方向，而非一套合并的信息架构拆分成 3 种视觉变体。

- 方向一：发现优先。
- 方向二：Spatial/List 合并首页，含集合侧栏。
- 方向三：激活优先。

只有方向二可以承载合并式 Spatial/List 框架加持久集合侧栏的模式。
