## 1. 文档目的

这份 PRD 用于生成 3 个并行的首页原型方向。

它不是一份单一整合方案的 PRD。

不应将其理解为：

- 一套主 IA 的 3 个视觉变体；
- 所有方案复用同一套全局导航与二级导航；
- 一个把所有方向混合在一起的合并首页概念。

它应被理解为：

- 3 个明确不同的首页方向；
- 3 个不同的首页核心命题；
- 3 个结构上不同的原型输出；
- 在共享业务约束下，给出不同的 IA 与布局回应。

## 2. 不可变的共享约束

所有方向都必须遵守以下产品事实：

- imToken Web 基于账户抽象；
- 不应暗示种子词设置流程；
- onboarding 应保持轻量；
- campaign 必须可以被营销团队运营；
- 优先使用现有 API 与低成本内容字段；
- 在现有能力允许范围内，增加 token 与 collection 信息密度；
- 当前阶段不依赖 AI insight；
- 当前阶段不依赖新的 personalization engine；
- 首页需要显著提升用户的探索动机。

所有方向共享的内容机会包括：

- token 数量；
- representative tokens；
- 若已有则展示价格与 24h 变化；
- chain / network；
- collection type / source；
- campaign label 与 CTA；
- passkey / 无 seed phrase 提示；
- gas abstraction 提示；
- Send by Link 提示；
- 轻量 trust / status 提示。

这些是共享约束，不代表必须使用同一套布局。

## 3. 并行方向规则

以下 3 个方向必须保持相互独立。

它们是并行概念方向，不是可以层层叠加的累计方案。

原型生成规则：

- 产出 3 个清晰不同的首页概念；
- 不要从一套基础线框出发，仅调整强调点；
- 不要在 3 个方向里复用同样的顶部导航加二级导航结构；
- 只有 Direction 2 可以把融合 Spatial/List 外壳与持续 collection 侧栏作为首页核心模式。

方向归属：

- Direction 1 负责 discovery 与 campaign 理解；
- Direction 2 负责融合双视角探索与 collection 侧栏连续性；
- Direction 3 负责 activation 与 onboarding 信心建立。

## 4. 用户与业务目标

### 首次访问用户目标

- 在第一屏内理解产品是什么；
- 理解无需先创建账户也可以浏览；
- 理解账户创建是轻量的；
- 获得足够信息来决定点击哪里。

### 回访用户目标

- 快速恢复探索状态；
- 理解当前有什么重点内容或变化；
- 在需要时能清楚进入自己的资产视图。

### 业务目标

- 让首页更适合 campaign 运营；
- 提升用户打开 collection 的动机；
- 更清楚地表达产品定位；
- 为未来 personalization 留空间，但当前不依赖它。

## 5. Direction 1：发现优先首页

### 核心命题

Direction 1 将首页首先定义为更强的 token discovery 与 campaign 承载界面。

这一方向需要回答：

- 为什么我要打开这个 collection？
- 当前重点内容是什么？
- 在创建账户前，我能理解哪些有价值的 token 信息？

### 核心 UX 意图

- 提升 collection 理解效率；
- 提升 campaign 清晰度；
- 提升点击动机；
- 让现有数据更有用。

### 必要元素

- 使用现有数据丰富 collection 卡片；
- 带有结构化标题、描述、CTA 与关联 collection/token 的 campaign 模块；
- 作为决策层的 collection detail；
- 能解释“为什么值得点开”的扫读级信息。

### 布局指导

Direction 1 可以使用：

- 一个主要 discovery 画布或 feed；
- 一个主浏览模式；
- campaign-led hero 或 spotlight 区域；
- 其下或其旁的 supporting collection 模块。

Direction 1 应避免：

- 以 `Market` / `My Holdings` 作为首页主框架；
- 把 `Spatial` / `List` 作为必须存在的二级导航对；
- 把 portfolio ownership 当作首页主要信息；
- 让 onboarding-first 的首屏块主导整个页面。

### 原型判断信号

- collection 之间的差异性更清晰；
- campaign context 无需额外解释也能理解；
- 用户在扫读层就获得点击理由；
- 首页更像 discovery，而不是 account setup。

## 6. Direction 2：融合 Spatial/List 首页

### 核心命题

Direction 2 将首页首先定义为一个融合式 discovery shell，其中 Spatial View 与 List View 是同一 collection system 的两种表达。

这一方向需要回答：

- 如何让 Spatial 与 List 像同一个首页，而不是两个割裂体验？
- 用户切换模式时，如何保留上下文？
- 浏览过程中，如何始终保持 collection 理解层可见？

### 核心 UX 意图

- 保留模式之间的连续性；
- 让 Spatial 不只是有表现力，也真正可用；
- 让 List 更密集，但不变成另一个独立页面；
- 让 collection 理解层成为持续存在的辅助层。

### 必要元素

- 一个共享的 Spatial / List 外壳；
- 紧凑的 Spatial / List 模式切换；
- 持续存在的 collection 侧栏或 selected panel；
- 首次进入时自动选中 campaign hub 或 featured collection；
- 同一个 collection object 在两种模式中渲染；
- List 模式复用同一个侧栏逻辑，而不是切换到单独 detail modal；
- 侧栏中能容纳 campaign context、token rows、CTA 与 account cue。

### 布局指导

Direction 2 可以使用：

- 一个全高共享外壳；
- 主浏览区与侧栏约 65/35 的分割；
- Spatial canvas 作为一种模式；
- List grid 或 list rows 作为另一种模式；
- 切换模式时保留已选 collection；
- 紧凑顶部 chrome，把视觉重心留给主内容区与侧栏。

Direction 2 应避免：

- 重新退回成两个分离的 Spatial 与 List 页面；
- 把侧栏仅当作临时 modal 使用；
- 模式切换后丢失选中上下文；
- 让 activation 成为页面的主叙事。

### 原型判断信号

- 切换模式像切换观察视角，而不是离开当前页面；
- selected collection panel 成为主要连续性层；
- 首屏就能在侧栏看到 campaign context；
- 用户无需进入单独 detail 流程，也能理解一个 collection。

### 当前 V2 融合模式需要保留的部分

基于当前原型材料，Direction 2 应继承以下 V2 模式：

- tabs 与 mode switch 压缩在一个紧凑的 shell 里；
- campaign hub 默认选中；
- 浏览区与 selected panel 维持 65/35 分割；
- collection 侧栏在 Spatial 与 List 两种模式下都持续存在；
- List 模式继续使用同一个 selected panel，而不是跳到单独 detail 页面；
- `My Holdings` 可以作为次级 tab 存在，但融合 discovery shell 才是主命题。

## 7. Direction 3：激活优先首页

### 核心命题

Direction 3 将首页首先定义为轻量 activation 与 onboarding 入口。

这一方向需要回答：

- 这个产品是什么？
- 为什么它的 setup 很轻？
- 为什么我现在应该创建账户？
- 创建账户后，下一步会发生什么？

### 核心 UX 意图

- 降低 setup 焦虑；
- 让账户抽象更具象；
- 把 signup 连接到第一步动作；
- 保持 onboarding 简短且情境化。

### 必要元素

- 清晰的主账户 CTA；
- 简洁的 AA / passkey 信心提示；
- 3 个 capability cues：passkey account、gas abstraction、Send by Link；
- 快速 15 秒 demo 或 storyboard；
- 清晰的 post-CTA 回流到有意义动作的路径；
- 1 到 2 个作为激活上下文的 featured campaign 或 collection。

### 布局指导

Direction 3 可以使用：

- 更强的激活型首屏；
- CTA + proof / demo 组合块；
- 位于 onboarding 信息下方或旁边的辅助浏览模块；
- 比 Direction 1 更简单的 browse surface。

Direction 3 应避免：

- 把广义 market taxonomy 作为首页主叙事；
- 继承 Direction 2 的融合 Spatial/List shell 与持续侧栏；
- 呈现为 portfolio dashboard；
- 把首页做成长篇教育页。

### 原型判断信号

- 产品看起来更容易开始使用；
- 15 秒 demo 能把 setup 具象化；
- account CTA 出现得合理，而不是太早；
- 首页更像 activation，而不是市场浏览器。

## 8. 跨方向区分规则

这些规则应在原型生成时严格执行：

- Direction 1 不应继承 Direction 2 的融合 Spatial/List shell 与持续侧栏；
- Direction 3 不应继承 Direction 2 的融合 Spatial/List shell 与持续侧栏；
- Direction 1 与 Direction 3 可以都包含 search 与 account entry，但不应复用 Direction 2 的融合 shell 模式；
- Direction 1 即使带有 onboarding cue，也应整体呈现为 discovery-first；
- Direction 2 即使带有 onboarding cue，也应整体呈现为 continuity-first 的融合双视角；
- Direction 3 即使带有 browse modules，也应整体呈现为 activation-first。

如果两个方向最终拥有同样的 global nav、同样的 sub-nav 与同样的页面骨架，那么本轮概念生成就算失败。

## 9. 原型交付物

生成 3 个独立原型：

1. `Prototype A` 对应 Direction 1
   - discovery-first；
   - 以 campaign 与 collection 理解为核心；
   - 不使用融合 Spatial/List + 持续 collection 侧栏作为主模式。

2. `Prototype B` 对应 Direction 2
   - 融合 Spatial/List 首页；
   - 以 collection 侧栏连续性为核心；
   - 延续当前 V2 shell。

3. `Prototype C` 对应 Direction 3
   - activation-first；
   - 以 passkey / 轻量 onboarding 为核心；
   - 不使用融合 Spatial/List + 持续 collection 侧栏作为主模式。

每个原型都应定义：

- 第一屏层级；
- 主 CTA 行为；
- campaign 放置方式；
- collection 信息密度；
- account-entry 处理方式；
- 匿名 / 登录状态逻辑（如适用）；
- 高层级的移动端适配。

## 10. 共享内容清单

所有方向都可以在已有条件下使用以下内容：

- collection title；
- 若已配置，则使用 collection description；
- representative token icons / names；
- token count；
- 若已有，则使用 token price 与 24h movement；
- chain / network；
- source / type label；
- campaign title、description、poster、CTA；
- 若可靠，则使用 updated time；
- passkey account 文案；
- gas abstraction 文案；
- Send by Link 文案。

## 11. 当前阶段不包含

任何方向都不应依赖：

- AI-generated insight modules；
- 新的 personalization logic；
- 新的 recommendation engine；
- advanced campaign targeting；
- 全面 portfolio analytics redesign；
- 重型 market-terminal controls；
- 长篇 onboarding 教育页。

## 12. 验收标准

当以下条件成立时，这份文档就算运作正确：

- 生成器产出 3 个结构差异明显的首页概念；
- 只有 Direction 2 使用融合 Spatial/List shell + 持续 collection 侧栏作为核心模式；
- Direction 1 明确呈现为 discovery-first；
- Direction 2 明确呈现为融合双视角 continuity-first；
- Direction 3 明确呈现为 activation-first；
- 3 个输出不会看起来像一套共享线框的不同强调版本。

## 13. 工作总结

这份 PRD 应驱动 3 个并行原型，而不是一个拆成多个变体的合并首页。

- Direction 1 是 discovery-first。
- Direction 2 是融合 Spatial/List + collection 侧栏。
- Direction 3 是 activation-first。

只有 Direction 2 应把融合 Spatial/List shell 与持续 collection 侧栏作为首页核心模式。
