## 1. Document Purpose

This PRD is for generating 3 parallel homepage prototypes.

It is not a single combined solution PRD.

It should not be interpreted as:

- one master IA with 3 styling variations;
- one shared global nav and sub-nav structure reused across all options;
- one merged homepage concept that mixes all directions together.

It should be interpreted as:

- 3 distinct homepage directions;
- 3 different homepage premises;
- 3 structurally different prototype outputs;
- shared business constraints with different IA and layout responses.

## 2. Non-Negotiable Shared Constraints

All directions must respect these product truths:

- imToken Web is account-abstraction based;
- no seed phrase setup should be implied;
- onboarding should remain lightweight;
- campaigns need to be operable by marketing;
- existing APIs and low-effort content fields should be preferred;
- richer token and collection info should be surfaced where already available;
- no dependency on AI insights in this phase;
- no dependency on a new personalization engine in this phase;
- homepage should create stronger motivation to explore.

Shared content opportunities across all directions:

- token count;
- representative tokens;
- price and 24h movement if available;
- chain/network;
- collection type/source;
- campaign label and CTA;
- passkey/no seed phrase cue;
- gas abstraction cue;
- Send by Link cue;
- lightweight trust/status cues.

These are shared constraints, not instructions to use the same layout.

## 3. Parallel Direction Rules

The 3 directions below must remain separate.

They are parallel concept tracks, not cumulative layers.

Prototype generation rule:

- produce 3 distinct homepage concepts;
- do not start from one base wireframe and tweak emphasis;
- do not reuse the same top navigation plus secondary navigation across all 3;
- only Direction 2 may use the merged Spatial/List shell with persistent collection sidebar as a core homepage pattern.

Direction ownership:

- Direction 1 owns discovery and campaign understanding.
- Direction 2 owns merged dual-lens exploration with collection sidebar continuity.
- Direction 3 owns activation and onboarding confidence.

## 4. User And Business Goals

### First-Time User Goals

- understand the product within the first viewport;
- understand they can explore before creating an account;
- understand account creation is lightweight;
- see enough information to decide where to click.

### Returning User Goals

- resume exploration quickly;
- understand what is featured or relevant now;
- access owned assets clearly when needed.

### Business Goals

- make the homepage more operable for campaigns;
- improve motivation to open collections;
- express product positioning more clearly;
- preserve room for future personalization without depending on it now.

## 5. Direction 1: Discovery-First Homepage

### Core Premise

Direction 1 treats the homepage primarily as a stronger token discovery and campaign surface.

This direction should answer:

- Why should I open this collection?
- What is featured now?
- What useful token information can I understand before account creation?

### Core UX Intent

- increase collection comprehension;
- increase campaign clarity;
- increase click motivation;
- make existing data feel more useful.

### Required Elements

- richer collection cards using existing data;
- campaign module with structured title, description, CTA, and linked collection/token;
- collection detail as a decision layer;
- enough scan-level information to explain why a collection matters.

### Layout Guidance

Direction 1 can use:

- one main discovery canvas or feed;
- one primary browse mode;
- campaign-led hero or spotlight area;
- supporting collection modules underneath or beside it.

Direction 1 should avoid:

- `Market` / `My Holdings` as the primary homepage frame;
- `Spatial` / `List` as a required sub-nav pair;
- portfolio ownership as a major homepage message;
- an onboarding-first opening block that dominates the page.

### Good Prototype Signals

- collections feel meaningfully different from one another;
- campaign context is clear without extra explanation;
- users get reasons to click at scan level;
- homepage feels more like discovery than account setup.

## 6. Direction 2: Merged Spatial/List Homepage

### Core Premise

Direction 2 treats the homepage primarily as one merged discovery shell where Spatial View and List View are two expressions of the same collection system.

This direction should answer:

- How can Spatial and List feel like one homepage instead of two disconnected experiences?
- How can users switch modes without losing context?
- How can collection understanding remain visible while browsing?

### Core UX Intent

- preserve continuity across modes;
- make Spatial useful, not only expressive;
- make List denser without turning it into a separate page;
- keep collection understanding visible as a persistent layer.

### Required Elements

- one shared shell for Spatial and List;
- compact mode switch between Spatial and List;
- persistent collection sidebar or selected panel;
- campaign hub or featured collection auto-selected on entry;
- same collection object rendered in both modes;
- list mode that uses the same sidebar logic rather than a separate detail modal;
- room for campaign context, token rows, CTA, and account cue inside the sidebar.

### Layout Guidance

Direction 2 can use:

- a full-height shared shell;
- a 65/35 or similar split between main browsing area and sidebar;
- Spatial canvas as one mode;
- List grid or list rows as the other mode;
- selected collection persistence while switching modes;
- compact top chrome so the canvas and sidebar remain the primary content.

Direction 2 should avoid:

- collapsing back into two separate pages for Spatial and List;
- treating the sidebar as a temporary modal only;
- losing selection context when the user changes mode;
- turning activation into the main narrative of the page.

### Good Prototype Signals

- switching modes feels like changing lens, not leaving the page;
- the selected collection panel becomes the main continuity layer;
- campaign context is visible in the sidebar on first load;
- users can understand a collection without opening a separate detail flow.

### Current V2 Merge Pattern To Carry Forward

Based on current prototype materials, Direction 2 should inherit these V2 patterns:

- tabs and mode switch compressed into a single compact shell;
- campaign hub auto-selected on open;
- 65/35 split between browse area and selected panel;
- collection sidebar persists in both Spatial and List modes;
- list mode uses the same selected panel instead of a separate detail destination;
- `My Holdings` may exist as a secondary tab, but the merged discovery shell is the main idea.

## 7. Direction 3: Activation-First Homepage

### Core Premise

Direction 3 treats the homepage primarily as a lightweight activation and onboarding entry point.

This direction should answer:

- What is this product?
- How easy is setup?
- Why should I create an account now?
- What happens immediately after account creation?

### Core UX Intent

- reduce setup anxiety;
- make account abstraction tangible;
- connect signup to first action;
- keep onboarding short and contextual.

### Required Elements

- clear primary account CTA;
- compact AA/passkey reassurance;
- 3 capability cues: passkey account, gas abstraction, Send by Link;
- quick 15s demo or storyboard;
- clear post-CTA route into meaningful action;
- one or two featured campaigns or collections as activation context.

### Layout Guidance

Direction 3 can use:

- a stronger activation-led first viewport;
- a CTA plus proof/demo block;
- supportive browse content below or beside the onboarding message;
- a simpler browse surface than Direction 1.

Direction 3 should avoid:

- broad market taxonomy as the primary homepage story;
- inheriting Direction 2's merged Spatial/List shell with persistent sidebar;
- a portfolio dashboard feel;
- turning the homepage into a long education page.

### Good Prototype Signals

- the product feels easy to start;
- the 15s demo makes setup tangible;
- the account CTA feels justified, not premature;
- homepage feels more like activation than a market browser.

## 8. Cross-Direction Separation Rules

These rules should be enforced during prototype generation:

- Direction 1 should not inherit Direction 2's merged Spatial/List shell and persistent sidebar.
- Direction 3 should not inherit Direction 2's merged Spatial/List shell and persistent sidebar.
- Direction 1 and Direction 3 may still include search and account entry, but not Direction 2's merged shell pattern.
- Direction 1 should feel discovery-first even if onboarding cues are present.
- Direction 2 should feel merged dual-lens and continuity-first even if onboarding cues are present.
- Direction 3 should feel activation-first even if browse modules are present.

If two directions end up sharing the same global nav, same sub-nav, and same page skeleton, the concept generation has failed the brief.

## 9. Prototype Deliverables

Generate 3 separate prototypes:

1. `Prototype A` for Direction 1
   - discovery-first;
   - campaign and collection understanding centered;
   - no merged Spatial/List shell with persistent collection sidebar.

2. `Prototype B` for Direction 2
   - merged Spatial/List homepage;
   - collection sidebar continuity centered;
   - current V2 shell carried forward.

3. `Prototype C` for Direction 3
   - activation-first;
   - passkey/lightweight onboarding centered;
   - no merged Spatial/List shell with persistent collection sidebar.

Each prototype should define:

- first viewport hierarchy;
- primary CTA behavior;
- campaign placement;
- collection information density;
- account-entry treatment;
- anonymous versus signed-in logic where relevant;
- mobile adaptation at a high level.

## 10. Shared Content Inventory

Any direction may use the following content inputs if they are already available:

- collection title;
- collection description if configured;
- representative token icons/names;
- token count;
- token price and 24h movement if available;
- chain/network;
- source/type label;
- campaign title, description, poster, CTA;
- updated time if reliable;
- passkey account copy;
- gas abstraction copy;
- Send by Link copy.

## 11. Out Of Scope

Do not make any direction depend on:

- AI-generated insight modules;
- new personalization logic;
- new recommendation engine;
- advanced campaign targeting;
- full portfolio analytics redesign;
- heavy market-terminal controls;
- long-form onboarding education page.

## 12. Acceptance Criteria

This document is working correctly when:

- the generator produces 3 structurally different homepage concepts;
- only Direction 2 uses the merged Spatial/List shell with persistent collection sidebar as its core pattern;
- Direction 1 reads as discovery-first;
- Direction 2 reads as merged dual-lens continuity-first;
- Direction 3 reads as activation-first;
- the 3 outputs do not look like one shared wireframe with different emphasis.

## 13. Working Summary

This PRD should drive 3 parallel prototypes, not 1 merged homepage broken into variations.

- Direction 1 is discovery-first.
- Direction 2 is the merged Spatial/List homepage with collection sidebar.
- Direction 3 is activation-first.

Only Direction 2 should use the merged Spatial/List shell plus persistent collection sidebar as the core homepage pattern.
